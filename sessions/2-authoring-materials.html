Session 2

Authoring AI-Friendly Materials

Course materials as plain-text markdown

Summary

Carson walks through keeping all course materials in plain-text markdown so agents can read and edit them directly. He covers Slidev for slide decks, mermaid diagrams, converting old decks, fact-checking, and PDF export.

  • Why markdown and agents love plain text -- everything in the repo (lectures, quizzes, notes) is markdown
  • Agents can dump all the text into context -- contrast with the old Google Slides workflow he paid a student to convert
  • Text diffs make changes easy to review -- the IntelliJ diff view shows exactly what an agent changed
  • Slidev intro and features -- free markdown slide system at sli.dev, code-friendly with syntax highlighting
  • Progress-bar plugin built by Claude -- a quality-of-life Slidev plugin he had Claude write
  • The Slidev workflow -- run slidev from the command line pointed at a lecture, present on localhost
  • Mermaid diagrams baked into Slidev -- flowcharts, sequence and state diagrams as text, no drawing tool
  • Converting old Google Slides or PowerPoint decks -- Claude reads those formats and converts to Slidev
  • Screenshot to slide live demo -- paste an image and Claude adds a matching mermaid slide
  • Consistent style with CLAUDE.md -- discipline the voice, still rewrite to keep it your own
  • Curriculum gap analysis against peer schools -- compare a class to Berkeley, MIT, or a top-10 syllabus
  • Fact-checking generated content -- have the agent verify truth claims via web search
  • PDF export with just -- just gen-pdfs renders every deck to PDF for Canvas

Why Agents Love Plain Text

  • No binary format to parse or round-trip
  • Every change is a readable line-level diff
  • The same file feeds the slides, the PDF, and the agent
  .md  -> slidev   -> slides
  .md  -> pandoc   -> PDF
  .md  -> agent    -> edits

Slides Are Markdown

  • Slidev turns markdown into a slide deck
  • https://sli.dev/
  • One file per lecture, one slide per ---
  • My preferred format: concepts on the left, code or diagram on the right

Consistent Style

  • The slide rules live in CLAUDE.md, not in your head each time
  • The agent applies them to every deck it touches
  • Change the rule once and regenerate
* Concepts left, code/media right
* Max three bullets per slide
* No filler slides (no "Pitfalls")
* Cover slide first, summary slide last
* Relative image paths: ./public/...
