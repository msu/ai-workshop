How the speaker uses AI to automate running a course: lecture prep, grade entry,
Canvas friction fixes, and a main vs student branch split that turns the agent
into a TA.

* [Automating lecture prep](https://youtu.be/sy7_FeHPPRo?t=32) -- gen-pdfs and gen-notes scripts produce lecture PDFs and notes.
* [One-page podium notes](https://youtu.be/sy7_FeHPPRo?t=67) -- a folded one-pager of lecture flow and demos to review before walking in.
* [Feeding observations back to Claude](https://youtu.be/sy7_FeHPPRo?t=127) -- notes on what ran long or got missed go back into Claude to suggest lecture changes.
* [Generating final grade entries](https://youtu.be/sy7_FeHPPRo?t=194) -- a script reads the Canvas CSV and builds a script to fill in grades.
* [Python that writes JavaScript](https://youtu.be/sy7_FeHPPRo?t=320) -- paste the generated JS into the Canvas grade-entry console; unmatched names are printed to spot check.
* [Pasting a YouTube link into Canvas](https://youtu.be/sy7_FeHPPRo?t=672) -- schedule a stream, copy the link, Ctrl-V auto-creates a titled Canvas link.
* [Tamper Monkey for per-site JavaScript](https://youtu.be/sy7_FeHPPRo?t=703) -- a browser extension running Claude-written scripts to fix Canvas pain points.
* [Drag-and-drop PDF upload](https://youtu.be/sy7_FeHPPRo?t=799) -- another Tamper Monkey script to drop a PDF straight into Canvas.
* [main vs student branch split](https://youtu.be/sy7_FeHPPRo?t=1016) -- main has solutions, student ships stubs as a template repo with no history.
* [claude.md as a TA](https://youtu.be/sy7_FeHPPRo?t=1109) -- the student branch ships a file telling Claude to act as a teaching aid, not write code.
* [Stanford reused the file](https://youtu.be/sy7_FeHPPRo?t=1233) -- the same ~50 line file, also shipped as agents.md for other tools.
* [Standardizing on VS Code](https://youtu.be/sy7_FeHPPRo?t=1418) -- discussion of a free, standard student environment and which model to pair with it.
